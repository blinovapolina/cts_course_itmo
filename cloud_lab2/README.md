# Отчёт по лабораторной работе №2

### Цель работы

Целью данной лабораторной работы является ознакомление с облачными сервисами (Amazon Web Services и Microsoft Azure) и классификацией потребления облачных ресурсов.

--- 

### Исходные данные

В ходе выполнения использовали результаты лабораторной работы №1 (классификация сервисов AWS) + слепок данных биллинга облачного провайдера в формате CSV + документация провайдеров Amazon Web Services и Microsoft Azure.
- https://docs.aws.amazon.com/
- https://learn.microsoft.com/azure/

### Ход работы

На первом этапе нужно было импортировать CSV-файл в табличный редактор: для этого применялся инструмент импорта «Данные → Из текстового / CSV-файла» с указанием разделителя символа ;.

На втором этапе для каждого сервиса была выполнена классификация в соответствии с иерархической моделью, разработанной в лабораторной работе №1. Для каждой строки данных были заполнены следующие классификационные атрибуты:

- IT Tower;

- Service Family;

- Service Type;

- Service Sub Type;

- Service Usage Type.

### Этапы выполнения работы

1. Изучить биллинговые данные облачного провайдера.

2. Сравнить названия сервисов и параметры их использования с официальной документацией провайдера.

3. Для каждой строки данных определить:
   - к какому облачному сервису относится данное потребление,
   - тип сервиса (вычисления, хранение данных, аналитика, машинное обучение и т.д.),
   - каким образом используется сервис.

4. Выполнить классификацию сервисов, сохраняя логику и структуру, использованную в первой лабораторной работе.

5. На основе полученных результатов сформировать единую кросс-провайдерную сервисную модель для анализа облачных сервисов без привязки к конкретному провайдеру.


### Результаты работы

В результате выполнения лабораторной работы:

- был обработан слепок биллинговых данных облачного провайдера;

- выполнена классификация сервисов по иерархической модели IT Tower / Service Family;

- сформирована единая сервисная модель, пригодная для сравнения облачных сервисов различных провайдеров;

- получен итоговый файл в формате Excel, готовый для дальнейшего анализа и хранения.
