## Цель работы

Целью данной лабораторной работы является ознакомление с облачными сервисами Microsoft Azure и классификацией потребления облачных ресурсов

---

## Исходные данные

В ходе выполнения лабораторной работы использовался файл с биллинговыми данными облачного провайдера Microsoft Azure в формате CSV (вариант 8), классификация которых выполнялась с использованием иерархической модели облачных сервисов, разработанной в лабораторной работе №1, а также официальная документация Microsoft Azure:
- https://learn.microsoft.com/azure/

## Ход работы

На первом этапе был импортирован CSV-файл в табличный редактор Excel с использованием инструмента  
**Данные → Из текстового / CSV-файла** и указанием разделителя `;`.

На втором этапе для каждой строки биллинговых данных была выполнена классификация сервисов Microsoft Azure в соответствии с иерархической моделью, разработанной в лабораторной работе №1 для AWS

Для каждой строки данных были заполнены следующие классификационные атрибуты:
- IT Tower  
- Service Family  
- Service Type  
- Service Sub Type  
- Service Usage Type  

---

## Теория

В рамках лр используется униврсальная модель классификации облачных сервисов, в ее основе лежат три базовых подхода предоставления облачных услуг:

<img width="997" height="489" alt="image" src="https://github.com/user-attachments/assets/9dc6afbc-cb13-47b0-b3d5-db5853b79db3" />

### IaaS (Infrastructure as a Service)

Модель, при которой облачный провайдер предоставляет базовые инфраструктурные ресурсы, а управление операционной системой и программным обеспечением выполняется пользователем

**Примеры сервисов в Azure:**
- Virtual Machines  
- Virtual Network  
- Load Balancer  

### PaaS (Platform as a Service)

Модель предоставления готовой платформы для разработки и развертывания приложений без управления инфраструктурой

**Примеры сервисов в Azure:**
- Azure App Service  
- Azure Container Instances  
- Azure Event Grid  
- Azure Service Bus  
- Azure IoT Hub  
- Azure Cognitive Services  
- Azure Machine Learning  

### SaaS (Software as a Service)

Модель предоставления готовых приложений, доступных через интернет, без необходимости управления инфраструктурой и платформой

**Примеры сервисов в Azure:**
- Dynamics 365 Customer Insights  
- Visual Studio / Azure DevOps

## Практическая часть

Практическая часть лабораторной работы заключалась в анализе биллинговых данных Microsoft Azure и сопоставлении используемых сервисов с универсальной моделью классификации, разработанной на основе AWS

Для каждой строки биллинга были анализированны следующие параметры:
- категория и подкатегория метра (`Meter Category`, `Meter Sub-Category`)
- название метра (`Meter Name`)
- потребляемый сервис (`Consumed Service`)

На основе этих данных определялось:
- к какому облачному сервису относится потребление
- к какому уровню иерархии (IT Tower и Service Family) относится сервис
- какой именно тип потребления учитывается в биллинге

Все сервисы Microsoft Azure были распределены по иерархии  
**IT Tower → Service Family → Service Type → Service Sub Type → Service Usage Type**,  
что позволило нам привести данные к единой, кросс-провайдерной модели

## Этапы работы

В ходе работы я классифицировала следующие сервисы Microsoft Azure:

- Virtual Machines — IaaS-сервис вычислительных ресурсов, включая виртуальные машины с доп. лицензиями (SQL Server, Red Hat, BizTalk, Java), а также зарезервированные экземпляры

- Virtual Machines Licenses — лицензии на прог. обеспечение, используемое в составе виртуальных машин

- Azure Container Instances — PaaS-сервис для запуска контейнеров, потребление учитывается по времени работы и используемым ресурсам

- Azure App Service (Web Apps) — PaaS-платформа для хостинга веб-приложений, включая вычислительные часы, SSL и бесплатные веб-сайты

- Azure Storage — сервисы хранения данных с различными типами хранилищ и операциями чтения/записи

- Azure CDN — сервис для передачи данных через сеть доставки контента

- Virtual Network — сетевой сервис для организации внутренней связности и маршрутизации ресурсов

- Load Balancer — сетевой сервис балансировки нагрузки, а также использование в составе Azure Batch и Azure Machine Learning

- Azure Batch — сервис пакетных вычислений и планирования заданий

- Azure Machine Learning — сервис для обучения, развертывания и эксплуатации моделей машинного обучения

- Azure SignalR Service — сервис обмена сообщениями и поддержки взаимодействия в реальном времени

- Azure Event Grid — сервис обработки событий и операций доставки сообщений

- Azure Service Bus — брокер сообщений для интеграции распределённых приложений

- Azure Media Services — сервисы кодирования, обработки и потоковой передачи медиа-контента

- Azure IoT Hub — сервис для обмена сообщениями и управления IoT-устройствами

- Azure Cognitive Services — набор API искусственного интеллекта и когнитивных сервисов

- Azure Active Directory B2C — сервис управления идентификацией и аутентификацией пользователей

- Azure Site Recovery — сервис аварийного восстановления и репликации виртуальных машин

- Dynamics 365 Customer Insights — SaaS-сервис для анализа и управления клиентскими данными

- Visual Studio / Azure DevOps — инструменты разработки, CI/CD и мониторинга приложений

- Azure Marketplace — сторонние сервисы и решения, подключаемые через маркетплейс Azure
